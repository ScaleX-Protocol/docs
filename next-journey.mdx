By building on **Pelagos’ cross-chain DAG execution layer**, ScaleX enables truly atomic, bridge-free cross-rollup operations.

# Cross-chain Order Submission

### Problem  
Liquidity and vault state fragment across chains — each rollup has different block time, latency, and data availability.  
This creates inefficiencies in yield updates, liquidity, and reliance on bridges.

### Solution  
Pelagos provides a **multichain DAG consensus + messaging layer** that lets ScaleX maintain a single canonical vault, even as users interact from different rollups.  
All chains connected to Pelagos share a consistent view of state, and cross-chain actions can be executed atomically.

### Example  
- The main ScaleX vault lives on **Arbitrum**.  
- A user on **Base** deposits through Pelagos.  
- Pelagos validators sequence and validate the transaction, then commit it to the shared state. :contentReference[oaicite:8]{index=8}  
- The vault is updated in “real time” (according to Pelagos’ DAG + snapshot model), and that state is accessible on every connected chain.

### Impact  
- **Unified liquidity:** Single vault, accessible from all Pelagos-connected chains.  
- **Consistent state:** No more fragmentation or stale vault data.  
- **Bridge-free execution:** No need for wrappers or trust-minimized bridging.  
- **Responsive UX:** Users transact locally, but state is globally consistent.

# MEV-Resilient Execution

### Problem  
Large orders or liquidations can be front-run across chains, especially when messages are delayed or reordered.

### Solution  
Pelagos’ sequencing + consensus model guarantees a canonical, finalized order of cross-chain actions. There’s no “bridge window” where MEV bots can reorder or exploit messages: once validated, actions execute deterministically across chains. :contentReference[oaicite:9]{index=9}

### Example  
- A big deposit or liquidation is submitted from **Base**.  
- Pelagos sequences and finalizes the message with validators.  
- The action is executed across the chain-connected vault **in one atomic step**, without giving bots a reorder or frontrun opportunity.

### Impact  
- **Fair execution:** No priority gas or frontrunning across chains.  
- **Stable yield:** Vault operations can’t be gamed.  
- **Secure UX:** Predictable, auditable state changes.
